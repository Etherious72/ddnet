# DDNet 项目逐行教学（核心文件）

本文档按“文件 -> 行号 -> 作用”讲解核心训练/测试链路。

---

## 1) `param_config.py`

- L1-L9: 文件头与说明。
- L15: `dataset_name`，选择数据集（决定尺寸、训练规模、路径子目录）。
- L17: `learning_rate`，优化器学习率。
- L18: `classes`，输出通道数（速度模型一般为 1 通道）。
- L19: `display_step`，训练日志打印频率（按 iteration）。
- L20: `model_type`，主训练脚本入口模型关键字。
- L21: `device_mode`，设备模式（`auto|cpu|gpu`）。
- L26-L27: 备注说明 `train_size/test_size` 在不同数据格式的语义。
- L29-L128: 按数据集分支设置：
  - `data_dim` 输入尺寸
  - `model_dim` 输出尺寸
  - `inchannels` 输入通道
  - `train_size/test_size` 训练测试规模
  - `firststage_epochs/secondstage_epochs/thirdstage_epochs` 课程学习三阶段轮数
  - `loss_weight` DDNet 双任务损失权重
  - `train_batch_size/test_batch_size` batch 大小
- L127-L129: 防御分支，数据集关键字无效时直接退出。

---

## 2) `path_config.py`

- L11: 导入参数，尤其依赖 `dataset_name`。
- L18-L21: 定义主目录、数据目录、结果目录、模型目录。
- L27: 根据 `dataset_name` 动态结果目录：`results/<Dataset>Results/`。
- L28: 动态模型目录：`models/<Dataset>Model/`。
- L29: 动态数据目录：`data/<Dataset>/`。
- L31-L38: 若目标目录不存在则创建，保证训练/测试前路径可用。

---

## 3) `func/device_selector.py`

- L4: `get_runtime_device(config_mode="auto")`，统一设备选择入口。
- L9: 读取配置值并标准化小写。
- L10-L11: 校验模式仅可为 `auto|cpu|gpu`。
- L13: 查询 CUDA 可用性。
- L15-L16: 若强制 `gpu` 但 CUDA 不可用，抛出明确错误。
- L18: `use_cuda` 判定逻辑：
  - `gpu` -> True
  - `auto` 且 cuda 可用 -> True
  - 其他 -> False
- L19: 生成 `torch.device`。
- L21: 返回 `(device, use_cuda, selected_mode)`。

---

## 4) `func/datasets_reader.py`

### 4.1 `batch_read_matfile(...)` (L17-L57)

- L17-L21: 函数签名，读取 `.mat` 批量数据。
- L34-L36: 预分配三类数组：
  - `data_set` 地震输入
  - `label_set` 速度标签
  - `clabel_set` 速度轮廓标签
- L38: 遍历样本编号。
- L41-L43: 读取 `seismic*.mat` 的 `data` 字段。
- L45-L46: 维度变换 `(H,W,C)->(C,H,W)`。
- L47-L48: 拷贝通道到 `data_set`。
- L51-L53: 读取 `vmodel*.mat`。
- L54: 写入速度标签。
- L55: 调 `extract_contours` 生成轮廓标签。
- L57: 返回 `(seismic, [velocity, contour])`。

### 4.2 `batch_read_npyfile(...)` (L59-L111)

- L59-L62: 函数签名，读取 `.npy` 批量数据。
- L75-L76: 用列表暂存多个文件块。
- L78: 遍历文件编号。
- L85-L87: 读取 `seismic*.npy`。
- L89: 加入 `dataset` 列表。
- L96-L98: 读取 `vmodel*.npy`。
- L100: 加入 `labelset` 列表。
- L102-L103: `vstack` 拼接为大批量数组。
- L105-L106: 预分配轮廓标签（默认按每文件 500 条样本估计）。
- L107-L109: 对每个样本/通道提取轮廓。
- L111: 返回 `(dataset, [labelset, conlabels])`。

### 4.3 `single_read_matfile(...)` (L114-L149)

- L133-L136: 构造单样本 seismic/vmodel 路径。
- L138-L141: 读文件并 reshape。
- L144-L145: 输入转为 `(C,H,W)`。
- L147: 速度图提取 Canny 轮廓。
- L149: 返回单样本三元组。

### 4.4 `single_read_npyfile(...)` (L151-L176)

- L164-L168: 构造 `.npy` 路径。
- L170: 按 `readIDs[1]` 取目标地震样本。
- L171: 取对应速度标签 `[...,0]`。
- L174: 生成轮廓标签。
- L176: 返回单样本三元组。

## 5) `model_train.py`

### 5.1 入口与建模 (`determine_network`)：L27-L92

- L27: 统一建模函数，训练/测试都可复用。
- L38-L41: 通过 `get_runtime_device(device_mode)` 决定 CPU/GPU。
- L45-L72: 根据 `model_type` 创建模型对象：
  - `DDNet/DDNet70/SDNet/SDNet70/InversionNet/FCNVMB`
- L80-L81: 如果给了已有模型路径，加载 checkpoint。
- L84-L88: GPU 时用 `DataParallel(...).cuda()`；否则 `.to(device)`。
- L90: Adam 优化器，学习率来自 `param_config.py`。
- L92: 返回模型、设备、优化器。

### 5.2 数据加载与课程学习样本构造 (`load_dataset`)：L94-L157

- L104-L107: 分 `.mat/.npy` 两条读取路径。
- L109-L111: OpenFWI 标签做 min-max 归一化。
- L113-L120: 根据数据集设置“中炮道”索引和阶段参数。
- L122-L133: Stage1 课程学习：
  - 一部分通道加噪
  - 一部分通道幅值放大
  - 一部分通道用中炮替代
- L134-L137: Stage2：所有通道都替换为中炮。
- L142-L145: 组装 TensorDataset（输入、速度、轮廓）。
- L146-L150: DataLoader（使用 `train_batch_size`）。

### 5.3 单阶段训练 (`train_for_one_stage`)：L159-L264

- L176: `step = int(train_size / train_batch_size)`，估算每 epoch batch 数。
- L177: `save_epoch = cur_epochs // save_times`，中间模型保存间隔。
- L182: 从模型参数读取真实设备，避免写死 CUDA。
- L191: 按 batch 迭代。
- L202-L204: 输入和标签统一 `.to(model_device)`。
- L207: `optimizer.zero_grad()`。
- L208: DDNet 损失对象 `LossDDNet`。
- L210-L215: 按模型分支计算 loss：
  - DD 系列：速度 + 轮廓双任务
  - SD 系列：仅速度 MSE
- L222-L223: NaN 保护。
- L226: 反向传播。
- L229: 优化器更新。
- L233-L235: 按 `display_step` 打印迭代日志。
- L243-L247: 统计并记录 epoch 级 loss。
- L257-L260: 达到保存间隔则写出 `.pkl`。
- L262: 每轮保存 loss 曲线到 `results_dir`。

### 5.4 三阶段课程训练 (`curriculum_learning_training`)：L266-L338

- L284-L291: Stage1（如启用）
- L303-L310: Stage2（如启用）
- L322-L329: Stage3（如启用）
- 每阶段都：
  - 重新构建/加载模型
  - 构建对应 stage 数据
  - 训练并保存中间模型
  - 清理内存（`gc.collect()`）

### 5.5 主入口：L340-L342

- L341: 调用 `curriculum_learning_training(model_type)`，`model_type` 来自配置。

## 6) `model_test.py`

### 6.1 测试集加载 (`load_dataset`)：L27-L55

- L36-L40: `.mat/.npy` 两条读取路径。
- L41-L44: OpenFWI 标签归一化。
- L51-L53: 组装 DataLoader（`test_batch_size`）。

### 6.2 批量测试 (`batch_test`)：L57-L138

- L68: 载入测试数据。
- L71: 通过 `determine_network` 加载模型到目标设备。
- L73-L76: 指标记录数组（MSE/MAE/UQI/LPIPS）。
- L81: LPIPS 模型放到同设备。
- L90-L91: 当前 batch 数据迁移到设备。
- L95-L103: 按 `model_type` 调对应前向。
- L109-L111: 预测与标签转 numpy。
- L115-L116: 使用真实 batch 大小遍历（避免最后一批越界）。
- L121-L124: 逐样本计算 4 个指标。
- L132-L138: 输出指标均值与耗时。

### 6.3 单样本测试 (`single_test`)：L140-L215

- L155: 加载模型。
- L157-L163: 读取单样本并按数据集类型归一化。
- L166: 创建同设备 LPIPS。
- L170-L174: 单样本输入转 tensor 并迁移设备。
- L179-L187: 按模型类型前向推理。
- L193-L199: 反归整与指标计算。
- L201-L203: 输出指标与单样本耗时。
- L206-L215: 可视化地震道和速度图。

### 6.4 主入口：L217-L235

- `batch_of_single=1` 走批量测试，否则走单样本测试。
