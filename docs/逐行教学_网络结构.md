# DDNet 项目逐行教学（网络结构）

本文档是第二册，专讲 `net/*.py`。

---

## 1) `net/DDNet.py`

### A. 基础模块

- L21-L50 `unetConv2`
  - 两层卷积块（Conv + BN + 激活），是 U-Net 的基本单元。
  - `is_batchnorm` 控制是否启用 BN。

- L56-L75 `unetDown`
  - 先过 `unetConv2`，再 `MaxPool2d(2,2)` 下采样。
  - 输出尺寸减半，通道通常增大。

- L81-L109 `unetUp`
  - 上采样（反卷积或双线性插值）。
  - 与编码器特征做 skip-connection 拼接（`torch.cat`）。
  - `F.pad` 用于对齐空间尺寸。

### B. DDNet 主体

- L114-L151 `DDNetModel.__init__`
  - `filters=[64,128,256,512,1024]` 是主干通道宽度。
  - 编码器：`down1~down4 + center`。
  - 双解码器：
    - `dc1_*` 预测速度值（回归）
    - `dc2_*` 预测边界轮廓（2 类）
  - `dc1_final` 输出 `n_classes`（通常 1）；`dc2_final` 输出 2 类。

- L152-L190 `DDNetModel.forward`
  - 编码提特征：`down1 -> down4 -> center`。
  - 两个 decoder 并行解码。
  - L177/L187：按 `label_dsp_dim` 中心裁剪，避免边界尺寸不一致。
  - 返回 `[速度输出, 轮廓输出]`。

### C. DDNet 损失

- L191-L223 `LossDDNet`
  - `criterion1 = MSELoss`（速度回归损失）
  - `cross = F.cross_entropy(...)`（轮廓分类损失）
  - `entropy_weight` 在运行时放到 `outputs2.device`，CPU/GPU 都兼容。
  - 总损失：`weights[0]*mse + weights[1]*cross`。

### D. SDNet（单解码版本）

- L228-L275 `SDNetModel`
  - 架构与 DDNet 编码部分相同，但只保留一个 decoder 输出速度图。
  - 训练时一般只用 MSE。

## 2) `net/DDNet70.py`

这个文件面向 `70x70` 输出分辨率，前端加入了地震记录压缩模块。

### A. 前置压缩模块

- L18-L48 `SeismicRecordDownSampling`
  - 先将输入沿高度方向插值到 `width*8`（L37-L38）。
  - 多层 `ConvBlock` 持续下采样并提通道。
  - 目的：把 `1000x70` 的地震记录压到更适配 U-Net 的紧凑特征。

### B. 通用模块

- L54-L83 `unetConv2`：与 DDNet.py 同类。
- L89-L108 `unetDown`：下采样模块。
- L109-L132 `unetUp`
  - 与 DDNet.py 不同点：这里直接插值到 `output_lim`（固定目标尺寸）。
- L133-L155 `netUp`
  - 无 skip 拼接，仅做上采样 + 插值到固定尺寸。

### C. Conv 辅助块

- L161-L182 `ConvBlock`
  - 更灵活卷积块（支持非方形卷积核），类似 InversionNet 风格。
- L188-L207 `ConvBlock_Tanh`
  - 末端 `Tanh` 输出归一化特征/图。

### D. 损失函数

- L210-L241 `LossDDNet`
  - 与 `DDNet.py` 一致：`MSE + 加权 CE`。

### E. DDNet70 主体

- L247-L284 `DDNet70Model.__init__`
  - 前置压缩：`pre_seis_conv`。
  - 编码器从 `down3` 开始（输入通道已是 32）。
  - 双解码器输出：
    - `dc1_final` 速度
    - `dc2_final` 轮廓

- L285-L320 `forward`
  - `compress_seis = pre_seis_conv(inputs)`。
  - 编码 `down3/down4/down5/center`。
  - 双 decoder 分支解码并返回双输出。

### F. SDNet70

- L326-L387 `SDNet70Model`
  - 与 DDNet70 共享前端和编码器。
  - 只保留一个 decoder，输出速度图。

## 3) `net/InversionNet.py`

这是对比模型，采用更“连续下采样 + 连续反卷积”的结构。

### A. 基础层

- L18-L30 `ConvBlock`：Conv + Norm + LeakyReLU，可选 Dropout。
- L32-L42 `ConvBlock_Tanh`：末端 Tanh 输出块。
- L44-L54 `DeconvBlock`：反卷积上采样块。

### B. 主体结构

- L57-L85 `InversionNet.__init__`
  - 编码端：`convblock1~8`，逐步降采样到 `1x1` bottleneck。
  - 解码端：`deconv1_1~deconv6`，逐步恢复空间分辨率。

- L87-L117 `forward`
  - L89-L102：编码过程（注释里给了每层形状）。
  - L105-L114：解码回升。
  - L115：`F.pad(..., [-5,-5,-5,-5])` 做裁剪，从 `80x80` 到 `70x70`。
  - L116：最终输出 `1x70x70`。

## 4) `net/FCNVMB.py`

FCNVMB 本质上是单解码 U-Net 风格基线。

### A. 基础模块

- L16-L48 `unetConv2`：双卷积块。
- L50-L73 `unetDown`：卷积 + 池化下采样。
- L75-L108 `unetUp`：上采样 + skip 拼接 + 卷积融合。

### B. 主体网络

- L110-L139 `FCNVMB.__init__`
  - 标准 4 层编码 + center + 4 层解码。
  - `final` 为 `1x1 conv` 输出速度图。

- L140-L158 `forward`
  - 编码：`down1 -> down4 -> center`
  - 解码：`up4 -> up1`
  - L156：按 `label_dsp_dim` 做中心裁剪对齐。
  - 返回单输出速度图。

---

## 5) 四个网络怎么选

- `DDNet`：双解码 + 课程学习，主方法。
- `DDNet70`：面向 `1000x70 -> 70x70` 的改进版（加前置压缩）。
- `InversionNet`：对比基线，连续编码-解码结构。
- `FCNVMB`：对比基线，U-Net 单解码结构。

---

## 6) 阅读建议（网络层面）

1. 先读 `DDNet.py`，理解双解码思想。
2. 再读 `DDNet70.py`，理解“前置压缩 + 70x70 适配”。
3. 对比 `FCNVMB.py`，看单解码基线差异。
4. 最后读 `InversionNet.py`，理解另一类架构风格。
