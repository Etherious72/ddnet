# DD-Net 本次会话变更说明（中文）

本文档整理了本次会话中已执行的主要改动与环境操作。

## 一、CPU/CUDA 兼容性改造

### `model_train.py`
- 新增 `import os`。
- 新增环境变量 `DDNET_FORCE_CPU` 支持：
  - 保留旧逻辑注释（仅 `torch.cuda.is_available()`）。
  - 新逻辑：仅在 CUDA 可用且未强制 CPU 时使用 GPU。
- `determine_network(...)` 中增加 CPU 分支：非 CUDA 时显式 `net_model.to(device)`。
- 训练循环的数据迁移统一为 `.to(model_device)`，仅在 CUDA 时使用 non-blocking。
- 保留旧 `.cuda(...)` 写法为注释。
- 字符串比较从 `is not ""` 修正为 `!= ""`。

### `model_test.py`
- 新增 `import os`（当前未使用，不影响运行）。
- `lpips` 对象迁移到同一设备：`lpips.LPIPS(...).to(device)`。
- 测试输入统一使用 `.to(device)`，替代纯 CUDA 分支。
- 保留旧 `.cuda(...)` 写法为注释。
- 批量评估循环改为使用真实 batch 大小 `pd_vmodel.shape[0]`，避免最后一个 batch 越界。
- 保留旧 `for k in range(test_batch_size)` 写法为注释。

### `inversionnet_train.py`
- 新增 `import os`。
- 新增 `DDNET_FORCE_CPU` 支持（保留旧逻辑注释）。
- CUDA 时 `DataParallel(...).cuda()`；CPU 时 `InvNet.to(device)`。
- 批数据迁移统一为 `.to(device)`。
- 保留旧 CUDA-only 写法注释。
- 字符串比较从 `is not ""` 修正为 `!= ""`。
- 当前调试参数改动：
  - `Epochs: 120 -> 2`
  - `TrainSize: 48000 -> 3`
  - `save_times: 12 -> 1`

### `fcnvmb_train.py`
- 导入更新为：`batch_read_matfile, batch_read_npyfile`。
- 新增 `import os` 与 `DDNET_FORCE_CPU` 支持。
- 增加 CPU 模型放置分支：`fcnNet.to(device)`。
- 批数据迁移统一为 `.to(device)`，保留旧 `.cuda(...)` 注释。
- 字符串比较从 `is not ""` 修正为 `!= ""`。
- 按数据集类型选择读取方式：
  - SEG 系列：`batch_read_matfile`
  - OpenFWI 系列：`batch_read_npyfile`
- 修复前向调用参数错误：
  - `fcnNet(images)` -> `fcnNet(images, model_dim)`
- 当前调试参数改动：
  - `Epochs: 100 -> 2`
  - `TrainSize: 1600 -> 2`
  - `save_times: 2 -> 1`

### `func/utils.py`
- `model_reader(...)` 增加 `map_location=device`，支持 CPU 加载 GPU 保存的模型。
- 旧写法 `torch.load(save_src)` 已保留为注释。

## 二、损失函数设备无关化

### `net/DDNet70.py`

- 同时修复网络中未定义类名问题：
  - `unetUp1/netUp1/unetUp2/netUp2`
  - 替换为已存在的 `unetUp/netUp`。
